# Συχνές Ερωτήσεις

## Ρύθμιση & Σύνδεση

<AccordionGroup>
  <Accordion title="Τα checkouts μου εμφανίζονται σε κατάσταση NEEDS_SETUP. Τι κάνω;">
    Το ακίνητο δεν έχει πλήρη ρύθμιση. Μεταβείτε στο **Properties**, επιλέξτε το ακίνητο και συμπληρώστε και τα τέσσερα πεδία:
    - **Elorus Contact** — λογαριασμός πελάτη στο Elorus
    - **Invoice Sequence** — σειρά αρίθμησης παραστατικών
    - **Κατηγορία Τέλους** — για το κλιματικό τέλος
    - **Elorus Organization** — το organization έκδοσης

    Μετά την αποθήκευση, τα checkouts αλλάζουν σε `READY` στον επόμενο συγχρονισμό (έως 5 λεπτά).
  </Accordion>

  <Accordion title="Το Hosthub API key μου έχει αλλάξει. Πώς το ανανεώνω;">
    Settings → **Hosthub Connection** → **Update API Key**. Εισάγετε το νέο key και πατήστε Validate. Ο συγχρονισμός συνεχίζεται κανονικά χωρίς απώλεια δεδομένων.
  </Accordion>

  <Accordion title="Μπορώ να έχω πολλά Elorus organizations υπό τον ίδιο λογαριασμό;">
    Ναι. Μεταβείτε στο Settings → **Invoicing Connections** → **Add Connection** για να προσθέσετε νέο API key Elorus. Κάθε ακίνητο μπορεί να αντιστοιχιστεί ανεξάρτητα σε διαφορετικό organization — χρήσιμο αν έχετε ακίνητα σε διαφορετικά ΑΦΜ.
  </Accordion>
</AccordionGroup>

## Αποτυχίες Αποστολής

<AccordionGroup>
  <Accordion title="Η αποστολή αποτυγχάνει με «Invalid contact». Τι σημαίνει;">
    Ο λογαριασμός επαφής που έχει οριστεί για το ακίνητο δεν υπάρχει ή δεν είναι ενεργός στο Elorus. Ελέγξτε στο Elorus → Επαφές αν η επαφή είναι ενεργή, και επαναλέξτε την από τις ρυθμίσεις ακινήτου.
  </Accordion>

  <Accordion title="Ένα checkout είναι σε κατάσταση SENT_WITH_ERRORS. Τι γίνεται;">
    Το τιμολόγιο και η απόδειξη εξόφλησης **εκδόθηκαν επιτυχώς**, αλλά το τέλος κλιματικής ανθεκτικότητας απέτυχε. Ελέγξτε:

    1. Αν η κατηγορία τέλους είναι σωστά ρυθμισμένη στο Properties
    2. Αν οι φορολογικοί κανόνες στο Elorus είναι ενημερωμένοι

    Επικοινωνήστε με τη support στο [support@etherly.app](mailto:support@etherly.app) αν το πρόβλημα επιμένει.
  </Accordion>

  <Accordion title="Μπορώ να ακυρώσω ένα παραστατικό που εκδόθηκε κατά λάθος;">
    Η ακύρωση παραστατικών γίνεται **απευθείας στο Elorus** — δεν υποστηρίζεται από το Etherly Sync UI. Μετά την ακύρωση στο Elorus, το checkout παραμένει σε κατάσταση `SENT` στο σύστημα. Επικοινωνήστε με τη support για επαναφορά κατάστασης αν χρειαστεί επανέκδοση.
  </Accordion>

  <Accordion title="Η αποστολή αποτυγχάνει επανειλημμένα. Τι ελέγχω;">
    Με τη σειρά:
    1. **Elorus API key** — ελέγξτε αν είναι ακόμα έγκυρο στο Settings
    2. **Invoice Sequence** — ελέγξτε στο Elorus αν η sequence είναι ενεργή
    3. **Elorus Organization** — βεβαιωθείτε ότι το organization επιτρέπει τον τύπο παραστατικού
    4. **Credits** — ελέγξτε το υπόλοιπό σας στο Billing

    Αν κανένα από τα παραπάνω δεν λύνει το πρόβλημα, επικοινωνήστε με τη support.
  </Accordion>
</AccordionGroup>

## Συγχρονισμός

<AccordionGroup>
  <Accordion title="Γιατί δεν συγχρονίζονται νέα checkouts;">
    Πιθανές αιτίες:
    - **Hosthub API key ληγμένο ή άκυρο** — ελέγξτε στο Settings → Hosthub Connection
    - **Νέες κρατήσεις χωρίς ημερομηνία αναχώρησης** — δεν εμφανίζονται ως checkout μέχρι να οριστεί checkout date
    - **Hosthub API timeout** — το σύστημα επαναλαμβάνει αυτόματα, αλλά αν επιμένει επικοινωνήστε με τη support

    Δοκιμάστε χειροκίνητο συγχρονισμό πατώντας **Sync Now** στο dashboard.
  </Accordion>

  <Accordion title="Μια κράτηση ακυρώθηκε στο Hosthub αλλά εξακολουθεί να εμφανίζεται. Γιατί;">
    Οι ακυρώσεις ανιχνεύονται στον επόμενο συγχρονισμό (έως 5 λεπτά). Μόλις εντοπιστούν, το checkout επισημαίνεται ως ακυρωμένο και αφαιρείται από την ουρά τιμολόγησης. Αν μετά από 15 λεπτά εξακολουθεί να εμφανίζεται ενεργό, δοκιμάστε χειροκίνητο **Sync Now**.
  </Accordion>
</AccordionGroup>

## Τέλος Κλιματικής Ανθεκτικότητας

<AccordionGroup>
  <Accordion title="Πώς λειτουργεί ο υπολογισμός για κρατήσεις που εκτείνονται σε δύο εποχές;">
    Αν μια κράτηση εκτείνεται μεταξύ χειμώνα (1 Νοεμβρίου – 31 Μαρτίου) και καλοκαιριού (1 Απριλίου – 31 Οκτωβρίου), το σύστημα:
    1. Υπολογίζει τις νύχτες ανά εποχή
    2. Εφαρμόζει τον αντίστοιχο συντελεστή για κάθε εποχή
    3. Δημιουργεί **ξεχωριστά παραστατικά** τέλους — ένα για κάθε εποχή

    Όλα γίνονται αυτόματα.
  </Accordion>

  <Accordion title="Τι γίνεται αν αλλάξουν τα ποσά του κλιματικού τέλους;">
    Τα ποσά αντλούνται από τους φορολογικούς κωδικούς του **Elorus** — δεν είναι hardcoded στο σύστημα. Αν αλλάξει η νομοθεσία, αρκεί να ενημερώσετε τα ποσά στο Elorus και το Etherly Sync τα ανακτά αυτόματα.
  </Accordion>

  <Accordion title="Δεν βρίσκω την κατηγορία τέλους του ακινήτου μου. Τι κάνω;">
    Οι κατηγορίες προέρχονται από τους φορολογικούς κωδικούς που έχετε ορίσει στο Elorus. Αν δεν εμφανίζεται η κατάλληλη κατηγορία, βεβαιωθείτε ότι έχετε δημιουργήσει τον αντίστοιχο φορολογικό κωδικό στο Elorus. Αν είστε αβέβαιοι για την κατηγορία, συμβουλευτείτε τον λογιστή σας.
  </Accordion>
</AccordionGroup>

## Αυτόματη Τιμολόγηση

<AccordionGroup>
  <Accordion title="Η αυτόματη τιμολόγηση δεν εκτελέστηκε χθες βράδυ. Γιατί;">
    Πιθανές αιτίες:
    - Δεν υπήρχαν checkouts σε κατάσταση `READY` τη συγκεκριμένη ώρα
    - Τελείωσαν τα credits — ελέγξτε το Billing
    - Ο λογαριασμός ήταν εκτός περιόδου χάριτος
    - Τεχνικό πρόβλημα στη διαδικασία (σπάνιο — επικοινωνήστε με τη support)
  </Accordion>

  <Accordion title="Μπορώ να αλλάξω ώρα αυτόματης τιμολόγησης;">
    Ναι, ανά πάσα στιγμή. Settings → **Auto-Invoicing** → αλλάξτε την ώρα → Save. Η αλλαγή ισχύει από την επόμενη εκτέλεση.
  </Accordion>
</AccordionGroup>

## Χρεώσεις

<AccordionGroup>
  <Accordion title="Πότε λήγει η δωρεάν δοκιμή μου;">
    Η δωρεάν δοκιμή διαρκεί **30 ημέρες** από τη δημιουργία του λογαριασμού. Η ακριβής ημερομηνία λήξης εμφανίζεται στο Settings → Billing.
  </Accordion>

  <Accordion title="Τι γίνεται αν τελειώσουν τα credits μου;">
    Λαμβάνετε **7 ημέρες περίοδο χάριτος** κατά τις οποίες η τιμολόγηση συνεχίζεται κανονικά. Μετά τις 7 ημέρες, η αποστολή μπλοκάρεται μέχρι να αγοράσετε νέα credits ή να ενεργοποιήσετε συνδρομή.
  </Accordion>
</AccordionGroup>

    - Το ακίνητο ή checkout που αντιμετωπίζει πρόβλημα
    - Το μήνυμα σφάλματος (αν υπάρχει)
    - Την ημερομηνία που παρατηρήσατε το πρόβλημα
  </Accordion>
</AccordionGroup>
